Integration Examples

See how easy it is to integrate DGS-Pay into your application

E-commerce Store

JavaScript React Node.js 
// Initialize DGS-Pay
const dgsPay = new DGSPay('sk_test_...');

// Create payment form
const paymentForm = DGSPay.createForm({
  style: {
    base: {
      fontSize: '16px',
      color: '#424770',
      '::placeholder': {
        color: '#aab7c4',
      },
    },
  },
});

// Mount form
paymentForm.mount('#payment-element');

// Handle payment
document.getElementById('submit').addEventListener('click', async (e) => {
  e.preventDefault();
  
  const {error} = await dgsPay.confirmPayment({
    elements,
    confirmParams: {
      return_url: `${window.location.origin}/success`,
    },
  });
});

SaaS Subscription

Python Django Subscriptions 
import dgs_pay

# Initialize client
dgs = dgs_pay.Client(api_key="sk_test_...")

# Create customer
customer = dgs.customers.create(
    email="customer@example.com",
    name="John Doe"
)

# Create subscription
subscription = dgs.subscriptions.create(
    customer=customer.id,
    items=[{
        "price": "price_1234567890",
        "quantity": 1
    }],
    payment_behavior="default_incomplete"
)

# Confirm payment
payment_intent = dgs.payment_intents.retrieve(
    subscription.latest_invoice.payment_intent
)

Frequently Asked Questions

Everything you need to know about getting started with DGS-Pay

Is there a free trial?

Yes! You can create a free account and start testing immediately. No credit card required for test mode. You only pay when you start processing real payments in live mode.

How quickly can I start processing payments?

You can start building your integration immediately after account creation. Most developers have their first payment working within hours. To go live with real payments, you'll need to complete our onboarding process and verification, which typically takes 1-2 business days.

What programming languages do you support?

We have official SDKs for JavaScript, Python, PHP, Ruby, Java, .NET, Go, and more. Our API is also RESTful, so you can use any programming language that can make HTTP requests.

Do you provide test card numbers?

Yes! We provide comprehensive test card numbers for different scenarios including successful payments, declined cards, 3D Secure authentication, and more. Check our documentation for the complete list.

What kind of support do you offer?

All accounts include access to our comprehensive documentation, code examples, and community forums. Professional and Enterprise plans include priority email support, and Enterprise customers get dedicated account management.
