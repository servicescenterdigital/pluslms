Quick Start

Get up and running with DGS-Pay in minutes. This guide will walk you through your first payment integration.

Beta Access
DGS-Pay is currently in beta. Contact us for early access to test your integrations.

1. Create an Account

Sign up for a free DGS-Pay account to get your API keys and start testing.

Create Free Account

2. Install the SDK

Choose your preferred programming language:

JavaScript 
npm install @dgs-pay/js-sdk

# or with yarn
yarn add @dgs-pay/js-sdk

# or with CDN
<script src="https://cdn.dgs-pay.com/js/v1/dgs-pay.min.js"></script>

3. Make Your First Payment

Here's a simple example of processing your first payment:

JavaScript 
const DGSPay = require('@dgs-pay/js-sdk');

const dgsPay = new DGSPay('sk_test_your_secret_key');

async function createPayment() {
  try {
    const payment = await dgsPay.payments.create({
      amount: 2500, // $25.00
      currency: 'usd',
      source: 'tok_visa',
      description: 'My First Payment'
    });
    
    console.log('Payment successful!', payment.id);
  } catch (error) {
    console.error('Payment failed:', error.message);
  }
}

createPayment();

API Reference

Complete reference for all DGS-Pay API endpoints, parameters, and responses.

Base URL

All API requests are made to the following base URL:

https://api.dgs-pay.com/v1

Authentication

Authenticate requests using your secret API key in the Authorization header:

HTTP Headers 
Authorization: Bearer sk_test_your_secret_key_here

Payments

Create and manage payments.

POST /payments Create Payment

Create a new payment.

Parameters

amount integer required Amount in cents
currency string required Three-letter ISO currency code
source string required Payment source token
description string optional Description of the payment

Example Request

curl -X POST https://api.dgs-pay.com/v1/payments \
  -H "Authorization: Bearer sk_test_your_secret_key" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": 2500,
    "currency": "usd",
    "source": "tok_visa",
    "description": "My First Payment"
  }'

Example Response

{
  "id": "pay_1234567890",
  "object": "payment",
  "amount": 2500,
  "currency": "usd",
  "status": "succeeded",
  "description": "My First Payment",
  "created": 1234567890,
  "source": {
    "id": "src_1234567890",
    "object": "source",
    "type": "card",
    "card": {
      "brand": "visa",
      "last4": "4242"
    }
  }
}

Error Handling

Learn how to handle errors and exceptions in your DGS-Pay integration.

Error Response Format

When an error occurs, DGS-Pay returns a JSON error object with details: 

{
  "error": {
    "type": "card_error",
    "code": "card_declined",
    "message": "Your card was declined.",
    "decline_code": "generic_decline",
    "param": "source"
  }
}

Common Error Types

card_error

Card-related errors such as insufficient funds or incorrect details.

invalid_request_error

Invalid parameters in the request (missing required fields, invalid format).

api_error

Internal server errors or temporary service issues.

authentication_error

Invalid or missing API authentication.
